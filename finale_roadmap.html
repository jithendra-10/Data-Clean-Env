4 The Training Script (Critical New Piece)

Fine-tune with TRL GRPO —
show the reward curve climbing

Why TRL GRPO, not PPO or SFT?
GRPO (Group Relative Policy Optimization) is what the OpenEnv ecosystem is built for. It runs without a critic model — so it works on a free Colab T4 GPU using Unsloth's 4-bit quantization. Fine-tune Llama-3-8B-Instruct (not 70B — too big for Colab) directly on DataClean-Env episodes. Show reward going from ~0.45 untrained → ~0.80+ trained.
# training.py — DataClean-Env GRPO Training Script # Run in Google Colab with T4 GPU + HF compute credits # Install: pip install unsloth trl openenv-core from unsloth import FastLanguageModel from trl import GRPOConfig, GRPOTrainer from dataclean.env import DataCleanEnv from dataclean.tasks import TASK_REGISTRY import torch, json # 1. Load model with Unsloth 4-bit (fits T4 GPU) model, tokenizer = FastLanguageModel.from_pretrained( model_name = "unsloth/Meta-Llama-3-8B-Instruct", max_seq_length = 2048, load_in_4bit = True, ) model = FastLanguageModel.get_peft_model( model, r=16, target_modules=["q_proj","k_proj","v_proj","o_proj"] ) # 2. Reward function — wraps DataClean-Env episode def dataclean_reward(completions, prompts, **kwargs): rewards = [] for completion in completions: env = DataCleanEnv() obs = env.reset(task_id="task_1", seed=42) total = 0.0 try: action_dict = json.loads(completion) from dataclean.models import DataCleanAction result = env.step(DataCleanAction(**action_dict)) total = result.reward + env.grade() except: total = -0.5 rewards.append(total) return rewards # 3. Dataset — episodes as prompts def build_dataset(n=200): import datasets env = DataCleanEnv(); samples = [] for i in range(n): task = ["task_1","task_2","task_3"][i % 3] obs = env.reset(task_id=task, seed=i) samples.append({"prompt": obs.to_prompt()}) return datasets.Dataset.from_list(samples) # 4. GRPO Training config config = GRPOConfig( output_dir="dataclean-grpo", num_train_epochs=3, per_device_train_batch_size=2, gradient_accumulation_steps=4, learning_rate=2e-5, num_generations=4, logging_steps=10, save_steps=50, max_new_tokens=256, ) trainer = GRPOTrainer( model=model, reward_funcs=dataclean_reward, args=config, train_dataset=build_dataset(), tokenizer=tokenizer, ) trainer.train() # Save to HuggingFace Hub model.push_to_hub("jithendra/dataclean-llama3-8b-grpo")

Likely Q&A questions — prepare these answers
"Why data cleaning specifically?"
Because it's the single most universal, expensive, and unsolved problem in enterprise data workflows. Unlike code review or email triage, it has a ground truth — either the data is clean or it isn't. That makes grading objective and training signal dense.
"What's the provenance mechanic for?"
It trains agents to be systematic rather than lucky. A trial-and-error agent that stumbles into a clean dataset won't get the provenance bonus because its ops log won't replay cleanly. Only a planned, reproducible sequence earns it.
"How does this generalise beyond your 3 tasks?"
Any tabular dataset can be dropped into the task generator. The action space and reward function are domain-agnostic. Adding a fourth task is one new Python function. The architecture scales to any number of cleaning scenarios.
"How does training improve the model?"
GRPO optimises the model to output actions that maximise the reward function — which is our deterministic grader. The model learns which operations to apply, in which order, at what confidence level. The reward curve shows measurable improvement across episodes.

Immutable ops log + full replay verification. Not present in any OpenEnv environment. Tests whether agents plan systematically rather than stumbling into correct answers.

Confidence calibration
Agents self-report confidence 0–1. Rewarded for accuracy, penalised for overconfidence. Trains agents that know what they don't know — a capability missing from every existing benchmark.

Task 3 gives each column a different corruption type. Agents can't apply one fix globally — must diagnose each column independently. Tests genuine analytical reasoning.
